# Typola · Product Vision

> 产品愿景与原则。v1.0 · 2026-04-17

---

## 1. 使命（Mission）

**让每个人都能像写 Word 一样，自然地写 Markdown。**

Markdown 是信息时代最重要的写作格式之一，但主流工具要么要求用户接受双栏（源码 + 预览）的不自然交互（VS Code、Mark Text），要么是私有商业软件（Typora）。Typola 想做的是：把"所见即所得"的 Markdown 写作体验，以免费、开源、可持续的方式还给所有用户。

---

## 2. 一句话定位

**Typola 是一款面向 Windows 桌面的、所见即所得的开源 Markdown 编辑器。**

- 打开即用，不需要配置
- 文件即本地 `.md`，零锁定、可被任何工具读写
- 覆盖 Typora 的主流功能：实时渲染、Mermaid、数学公式、代码高亮、多格式导出
- MIT 开源，任何人可 fork、修改、商用

---

## 3. 目标用户画像

### 3.1 主要用户（Primary）

**"技术写作者"**
- 开发者、技术 blogger、开源项目维护者
- 写 README、文档、博客、技术笔记
- 需要代码高亮、Mermaid、LaTeX、表格、任务清单
- 过去用 Typora，现在因为付费 / 商业许可问题在找替代

**"知识型学生与研究者"**
- 本科生、硕博、学术研究者
- 写读书笔记、论文草稿、研究日志
- 重度使用数学公式与图表
- 对文件可移植性、可版本控制敏感（Git + Markdown）

### 3.2 次要用户（Secondary）

**"非技术写作者"**
- 产品经理、设计师、独立博主
- 对 Markdown 有基础认识，但怕看到源码
- WYSIWYG 模式降低门槛，让他们也能进入 Markdown 生态

### 3.3 明确不服务的用户（Non-Goals）

- 不面向 **富文本编辑**（不是 Word / Google Docs 的替代）
- 不面向 **实时协作团队**（v1 不做，v2 考虑）
- 不面向 **纯 Markdown 源码党**（VS Code / Vim 已经很好了）
- 不面向 **移动端用户**（v1 纯 Windows 桌面）

## 4. 核心价值主张

### 4.1 用户能得到什么（Value Proposition）

**"一款永远免费、永远本地、永远极简的 Typora 替代品。"**

| 对比维度 | Typora | VS Code | **Typola** |
|---|---|---|---|
| 所见即所得 | ✅ | ❌（双栏） | ✅ |
| 价格 | $14.99 | 免费 | **永久免费** |
| 开源 | ❌ | ✅ | ✅（MIT） |
| 配置成本 | 低 | 高 | **低** |
| Mermaid / LaTeX | ✅ | 需插件 | ✅（内置） |
| 多格式导出 | ✅ | 需插件 | ✅（HTML/PDF/docx） |
| 数据所有权 | 本地 | 本地 | **本地 + 可移植** |
| 厂商风险 | 商业停更风险 | 微软绑定 | **社区治理** |

### 4.2 对 Typora 用户的迁移价值

- 文件格式 100% 兼容（都是标准 Markdown）
- 快捷键与默认主题尽量对齐 Typora 习惯
- 学习成本 < 5 分钟
- 不需要放弃任何已有的 `.md` 文件

## 5. 设计原则（Principles）

产品决策遇到分歧时，按以下优先级判断：

### 5.1 本地优先（Local-first）
用户的数据 **永远**存在本地，以标准 `.md` 明文存储。任何功能都不应要求云账号。未来做同步也只是可选增强，不是前提。

### 5.2 克制优于功能堆砌（Restraint over Feature Creep）
能不加的功能就不加。每个功能进 v1 都需要满足"是 Typora 合格替代品"的标准，而不是"我想要"的标准。高级功能留给 v2 + 插件机制。

### 5.3 写作优于编辑（Writing over Editing）
Typola 的首要场景是 **持续长时间写作**，不是临时扫一眼源码。默认视图、字体、间距、快捷键都为"两小时连续写作"优化，而不是"随手改个文档"。

### 5.4 所见即所得是底线（WYSIWYG is the Floor）
在任何需要权衡的场景，WYSIWYG 体验优先于实现便利。可以用 1 个月解决"粘贴图片"的复杂度，不能把它做成"弹窗输入 URL"。

### 5.5 开源优于品牌（Open over Owned）
Typola 不做闭源商业版，不设 Pro 账号，不做 VIP 功能。任何商业化尝试（若发生）都通过可选服务（如托管同步）而非功能锁。

### 5.6 纯黑白（Monochrome by Default）
UI 使用纯黑白，不引入品牌强调色。这既是美学选择，也是一种态度：**不想抢走写作的注意力**。

### 5.7 面向工程师的开源质量（Shipped Like Engineers）
代码质量、类型完备性、测试覆盖、文档完善——把 Typola 做成一个值得 fork、值得贡献的项目，不是"能跑就行"的脚手架。

## 6. 差异化（Differentiation）

### 6.1 相对于 Typora
- **免费、开源、MIT**；无商业许可焦虑
- **社区治理**，不会因公司决策停更
- 视觉语言清晰的**纯黑白路线**，与 Typora 的"黑底+橙圆"拉开距离
- 未来插件机制允许社区扩展，Typora 没有

### 6.2 相对于 VS Code / Obsidian
- **默认 WYSIWYG**，不是双栏或后台 Markdown
- 专注**单文档长文写作**，不是知识图谱或工程项目
- 默认外观即成品，不需要装 50 个插件调教

### 6.3 相对于 Mark Text / Zettlr
- 更**专注** Typora 风格，不做双模式妥协
- 图标与视觉语言更有辨识度（Trio W 抽象标识）
- 中文/跨语言默认体验更好（字体回退、快捷键、菜单）

## 7. 产品边界（Scope）

### 7.1 v1 覆盖（一次发布全量）

见 `Markdown编辑器-规划文档.md` 第 2.1 节。核心模块：

- 文件与工作区（多标签、会话恢复、外部监听）
- 基础 Markdown 编辑（全量 GFM + 脚注 + Front Matter）
- 进阶渲染（Mermaid 双击编辑、KaTeX、Shiki）
- 视图（大纲、全局搜索、焦点/打字机模式）
- 导出（HTML、PDF、docx、PNG、Mermaid SVG/PNG）
- 主题系统（4-5 套内置 + 用户主题热加载）
- 设置、快捷键、i18n、更新机制（安装版自动更新 / 便携版版本检查下载）

### 7.2 v2 考虑

插件机制、Git 集成、图床、拼写检查、AI 写作、实时协作、同步、跨平台

### 7.3 永不做

- 富文本编辑（彩色字体、艺术字、自由排版）
- 商业锁定 / Pro 订阅
- 广告与数据收集
- 强制登录

## 9. 品牌叙事（Brand Story）

> Typora 很好，但它关闭了。于是我们写了一个它。
>
> 它叫 Typola。三个 W 代表 WYSIWYG —— What You See Is What You Get。
>
> 它只做一件事：把所见即所得的 Markdown 写作体验，以开源、永远免费、只属于你的方式交到你手里。
>
> 它是纯黑白的。因为我们希望你专注写字，而不是看着软件本身。

可用于 README 开头、官网 Hero、发布博文。

---

## 10. 长期愿景（3 年）

**做成一款"Markdown 写作的新默认"。**

- 在中文技术社群里成为 Typora 的默认继任者
- 在英文开源社区成为"免费且开源"选项里的首选（超越 Mark Text、Typora 替代品）
- 建立一个**可持续的开源治理模式**，不依赖单一作者持续投入
- 沉淀出一套**通用的 Markdown WYSIWYG 渲染引擎**（基于 Milkdown 扩展），可被其他项目复用
- 可选的托管服务（同步、协作、AI）作为收入来源，但永远不影响核心产品的免费开源属性
