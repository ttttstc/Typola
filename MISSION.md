# MISSION — Typola 使用教程

## 为什么写这套教程

Typola 是给**中文写作者**(产品经理 / 技术布道 / 独立研究员 / 公众号 / 日报周报)的本地 Markdown 工作台,核心是「文档是产物,AI 是工序」。这套教程面向**已经装好 Typola、准备真用起来**的人,目标是:

- 5 分钟内跑通浮条润色一个段落
- 一小时内能独立完成「写作 → AI 改稿 → 检视 → 导出 PDF」一个完整回合
- 遇到 AI 输出不确定怎么进下一步时,能立刻判断选哪条路

## 谁的目标用户不是这套教程

- 想知道 Typola 源码、贡献者想了解架构 —— 教程不替代 `docs/ARCHITECTURE.md`
- 想做 Skill / 命令面板开发者 —— 教程不替代 `docs/AI_WORKBENCH_SKILL_OS.md`
- 还在选型、想比较 Typola vs 其他工具 —— 这套教程假设你已经定了

## 教程的 4 个核心能力锚点

| 能力 | 是什么 | 默认在哪里 |
|---|---|---|
| **浮条快速 AI** | 选中一段 → 浮条出现 → 选动作 → **原地对比卡** → 采纳 | 编辑器选区上方 |
| **AI 工作台(心流模式)** | 左侧栏切到 AI 工作台,跟 Claude Code / OpenCode 多轮对话 | Shift+A |
| **Skill 场景** | 右侧栏场景卡(报告 / PPT / HTML) → 选 skill → 新会话 | 右栏 SkillHub |
| **检视模式** | 整篇加意见,「发 AI 改稿」走内联 diff 后回写编辑器 | 顶部模式切换 |

## 教程边界

- **包含**:装好后第一天的所有实操;4 个能力的真实使用流程;AI 输出不确定时的判路指引。
- **不包含**:CLI 安装、AI 账号申请(M1 第 0 课指引后跳过)、源码层架构、Skill 自开发、打包发布、自定义主题排版细节。

## 完成判定

学完这 10 课后,你能:
1. 用 3 种不同的方式让 AI 改一段文档,**且能选哪种更适合当下场景**。
2. 把一篇万字稿的 AI 审阅流程在 30 分钟内走完且不丢原文。
3. 解释「为什么 Typola 把产物放回中间编辑器、不放右边」。
4. 在没网络/没 AI CLI 的情况下,Typola 还能做什么、不能做什么。
