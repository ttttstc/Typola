第 0 课

Typola 第一天:装好就能写

5 分钟。装好、打开一份文档、写一段字、保存。所有后面的课都建立在这节。

这 9 节课会带你看懂的 5 大能力

解决什么

写作者每天的 4 个真痛点:(1) 写一段话要润色,**但不想离开文档去 ChatGPT 复制粘贴**;(2) 想让 AI 帮生成报告 / PPT,但结果困在聊天记录里,不能落到文件;(3) 整篇改稿时怕 AI 一次覆盖丢原文;(4) 同事让你 review,你不知道意见给到哪段、最后怎么合并回正文。

Typola 怎么做

所有 AI 调用都在文档内或文档旁发生;AI 写的东西落到本地文件而不是云端 chat;改稿前先给你逐处对比再合并;检视意见是叠加的一层,导出前不污染原文。

跟别家不同

Typora 不带 AI;Notion AI 是云文档,产物在云上你看不见本地 Markdown;Typola 把"AI 是工序,文档是产物"这件事死守住 — 所以左栏是草稿(A 对话),中栏是成品(C 文档),从不把对话变档案库。

1. 装哪一版 2. AI 命令行要不要先装 3. 打开你的第一份文档 4. 写一段再保存 5. 认识三种文档模式 试一试 陷阱

1. 装哪一版

Typola 目前只在 Windows 上发布。从 GitHub Releases 拿,两种包二选一:

安装版 Typola_*_x64-setup.exe

装在用户目录,自动关联 .md / .markdown / .html / .docx。日常使用选这个。 注意:Typola 也提供 .msi 格式;装到 Program Files 会弹管理员权限,大部分人不必用 msi。

免安装版 Typola_*_windows-x64_portable.zip

解压后双击 Typola.exe。适合临时测试、不想留安装痕迹、需要 U 盘带走的场景。 第一次启动会自动跑 WebView2 bootstrapper — 缺 WebView2 时会先装,无网就报错。

装好后第一次启动会做一次 WebView2 Runtime 预检。失败会弹窗告诉你去哪里装 — 跟着链接点过去装一次就好,以后不再弹。

快捷入门 4 步
  1. 装好 Typola
  2. 打开一份 .md 文档(后面第 3 步会教你方式)
  3. 默认就是阅读模式(所见即所得),不需要切
  4. 写完按 Ctrl + S 保存

2. AI 命令行要不要先装

看完第 0 节你就能写、就能保存。AI 那一大块要等本机有 Claude Code 或 OpenCode 之后才用得起来 — 后面的课都会用到。

如果你已经装了其中任一个,在 PowerShell 里跑一下确认:

claude --version
opencode --version

输出类似 claude 1.x.x 之类就行,装到具体哪个版本不重要。没装不影响第 0 课,但第 1 课开始就要至少一个能用了。

3. 打开你的第一份文档

四种打开方式,挑顺手的:

  1. 键盘 Ctrl + O → 选文件
  2. 拖拽:把 .md 文件从资源管理器拖到 Typola 窗口任意空白区
  3. 文件关联:装好 Typola 之后,双击 .md 默认就是它打开
  4. 命令行定位:Typola 启动时会读取最近的打开记录,直接出现在标题

几个常见文件类型都支持:

  • .md / .markdown / .mdx — Markdown
  • .html / .htm — 进入阅读/编辑链路
  • .docx — 只读预览(不要在 Typola 里改 Word,改完另存为 .md)
找不到合适的文档?

Typola 启动界面右侧的"快速打开"(Ctrl + Shift + P)会列出最近打开过的文件和当前常用工作目录,任选一个往下做。

4. 写一段再保存

默认就是阅读模式(WYSIWYG),你看到什么就是最终阅读的样子。可以直接:

  • 输入一段普通文字
  • 用 # 起标题、- 起列表、``` 起代码块 — 输完空格就立刻渲染
  • 图片可以直接粘贴剪贴板截图,Typola 会在文档同目录的 assets/ 下保存一份,并自动插入引用

第一次写完一定要按 Ctrl + S 保存。标签栏的文件名旁边会出现小圆点表示脏改,保存后小圆点消失。

Ctrl + S
保存到当前文件
Ctrl + Shift + S
另存为(可换目录、改后缀)
Ctrl + F
在当前文档里查找
Ctrl + H
查找并替换
Ctrl + Shift + P
快速打开(文件 + 最近记录)
关于"自动保存"

Typola 默认关闭自动保存(这是经过考虑的选择,不是 bug)。要打开去 设置;但前几课建议就用 Ctrl + S 手动保存,养成"改一段就存一段"的肌肉记忆,后面跟 AI 协作时才不会因为意外覆盖丢原稿。

5. 认识三种文档模式

Typola 同一篇文档,可以在三种模式之间切换 — 顶部那条三段控件就是切换器:

阅读模式(默认,WYSIWYG)

边写边看到接近终稿的排版。适合 90% 的写作时间。第一周你只用这一种就好。

心流模式

左侧栏让位给 AI 工作台;你可以一边编辑一边让 AI 帮忙。第 2 课展开。

检视模式

进入"给文章提意见"的节奏,不在原文写脏字,而是叠加一层意见;最后导成 review.md 给协作者看,或一键让 AI 按意见改稿。第 5 课展开。

三者的键盘切换:

Shift + A
心流 ↔ 阅读来回切
顶栏三段控件
任意两个模式之间点选

试一试

5 分钟任务:写一份简单的笔记
  1. 按 Ctrl + O 新建一个文件,命名为 my-first-note.md,放在桌面。
  2. 写一段不超过 200 字的自我介绍,加一个 # 标题 和 一个 - 列表。
  3. 在文档里粘一张图(随便截图都行),按 Ctrl + S 保存。
  4. 打开桌面上的 my-first-note 同目录,确认有 assets/ 文件夹,里面就是刚才那张图。

✅ 全部通过 = 你可以进入第 1 课:用浮条让 AI 在你眼皮底下改稿。

❌ 第 3 步图片没出现 / 文件没保存 / 找不到 assets/:去 陷阱区对照一下。

陷阱

  • 文件是 .docx 直接在 Typola 里改 — Typola 把它当只读预览,改了等于丢。开 Ctrl + Shift + S 另存为 .md,改 .md 才安全。
  • "自动保存开着怎么还是不存" — 默认就是关的。先去 设置 里勾上。
  • Ctrl + S 没反应 — 文档是从 Typola 之外"刚刚拖进来"的状态,会被当成未命名 — 看标题栏有没有 * 或文件名是不是 未命名。
  • WebView2 启动报错 — 跟着弹窗提示去 microsoft.com/webview2 装;装完重启 Typola。
