第 1 课

用浮条让 AI 在你眼皮底下改稿

5 分钟。第一次让 Typola 里的 AI 改一段话 — 选 → 浮条 → 原地对比 → 采纳。

能力定位

解决什么

"这段写得不好,但我懒得去 ChatGPT 复制粘贴" — 写一段就被打断一次的痛。 Typola 把 AI 改稿压成一条浮在选区上方的横排工具条,不离文档、不开新窗口。

Typola 怎么做

选中文字后延迟 160ms 自动出现的 SelectionFloatingBar。点动作 → 起一个隐藏的 AI 会话(只为了拿替换文本)→ 把"原文 vs 新版"叠回选区下方一张结果卡 → 你点 采纳替换 才真的写回正文。

跟别家不同

VS Code Copilot 是字符级补全(改你还没写的字);Notion AI 是整页 AI 改稿(一键覆盖,不可拒)。 Typola 走"选段级 + 原地对比 + 逐处采纳"比 Copilot 改得多,比 Notion 改得可控。 右键菜单 Ctrl + K 还有 4 个动作(缩写/扩写/校对/自定义)不在浮条里 — 浮条只挑了最常用的 3 个。

前置

  • Typola 已装,且至少一个 claude / opencode 可用
  • 设置 → AI 执行 里能看到 CLI 检测通过(版本号不是空的)
  • 编辑器里有一份写了一半的文档(自己随便写一段就好,不需要写得好)
CLI 没检测到?

常见原因:PowerShell 的 ExecutionPolicy 挡了 npm 的 .ps1,或装到了用户目录但 PATH 里没有。 最简单的兜底:在 设置 → AI 执行 → 可执行路径 里手动指定完整路径(例如 C:\Users\你\.npm-global\claude.cmd)。 改完点一下"检测",看到版本号就 OK。

1. 浮条润色 — 3 步让 AI 改完并采纳

这是最常用的动作,适合大部分"这段能更好"的场景。

步骤 1
在文档里选中一段中文/英文文本(鼠标拖选 3-5 句话,空段落、整段都行)
步骤 2
松开鼠标 ~160ms 后,选区上方会出现一条小工具条:「润色 / 解释术语 / 加检视意见」
步骤 3
点「润色」→ 选区下方贴出一张对比卡「原文 vs 新版本」+ 采纳替换 / 取消

点 采纳替换:原文被精确替换,自动加进撤销栈(Ctrl + Z 能撤回)。 点 取消:什么都没发生,文本还是你的。

采纳的底线逻辑

浮条不偷你文字。原文永远在磁盘的当前文档和撤销栈里 — 哪怕点了「采纳替换」,Ctrl + Z 也能一步步回退。 对比卡是叠加的,采纳前你看得到「会变成什么」。

2. 浮条只有 3 个动作?右键还有 4 个

浮条故意只挑了最高频的 3 个(润色 / 解释术语 / 加检视意见)。 其余的 4 个 — 缩写、扩写、校对、自定义 — 都在 右键菜单 / Ctrl + K 里:

  1. 选中一段文字
  2. 按 Ctrl + K 或右键 → AI 动作
  3. 在列表里选「缩写 / 扩写 / 校对 / 自定义」
  4. 弹出的输入卡里可以填具体要求(如"改得更口语","目标长度 80 字"),Ctrl + Enter 发送
  5. 同样回一张「原文 vs 新版本」对比卡,采纳替换或取消
什么时候用哪个?

润色 — 文字通顺但不够精炼,要打磨。
缩写 — 长了,要砍一半。
扩写 — 太干,要补论据/例子。
校对 — 怀疑有错别字/语法/标点问题,只想扫一遍不改写。
解释术语 — 选了一个词/概念想加一段人话注解(不替换,只展示),你可以复制走。
自定义 — 上面的都不合适,自己写要求(例如"翻成英文,保留技术名词")。

3. 解释术语 — 这是唯一一个"不替换"的动作

「解释术语」的特殊之处在于:结果不打算替换你选中的词,只是解释一下给你看。 所以它的结果卡显示成单文本块 + 复制按钮,没有"采纳替换":

典型用法:文档里有个专业术语,你写作时需要看一眼定义但不想让文档里多出这段解释。 点解释 → 复制走 → 关卡 → 文档不变。

浮条只有 3 个 — 加检视意见要等第 5 课

浮条第 3 个按钮「加检视意见」是给检视模式铺路的:它不会调 AI,只是给选中段落记一条意见。 真正发 AI 改稿的入口在 右侧检视面板,要切到检视模式才出现。第 5 课再展开。 这一课先只用「润色 + 解释术语」。

试一试

10 分钟任务:让 AI 改 3 段
  1. 在自己的笔记文档里选一段长句子(超过 50 字),点浮条「润色」,看新版本,采纳。
  2. 再选一段,按 Ctrl + K 在右键菜单里选「缩写」,写要求"砍到一半长度",采纳。
  3. 选一个你不熟的术语,点浮条「解释术语」,看结果,点「复制」,粘贴到文档里随手位置,再撤销粘贴(练一下 Ctrl + Z)。
  4. 最后按 Ctrl + Z 几次回到最原始的版本 — 确认所有 AI 改动都能回退。

✅ 4 步全过 = 进入第 2 课:AI 工作台开箱。

陷阱

  • 浮条不出现 — 最常见是文档在"源码模式"(不是阅读模式),或你刚松开鼠标在拖选过程中。 等半秒别动鼠标,或在阅读模式(WYSIWYG)里选。
  • 浮条出现但点不动按钮 — 鼠标落在了按钮之间的空隙上。点按钮中央的图标或文字。
  • 点了「润色」卡在 loading 转圈 — CLI 进程没起来,或网络/鉴权问题。 等 8 秒还没结果会自动跳到错误卡,点「重试」,再不行查 设置 → AI 执行 的检测。
  • 采纳替换后文本看起来没变 — 通常是 AI 的"新版本"几乎等同原文(就是只换了几个字)。 退回看对比卡前后对比。
  • Ctrl+Z 不能回退 AI 改动 — 在阅读模式下是支持的。如果不行,先切到阅读模式再试。
