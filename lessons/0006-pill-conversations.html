第 6 课

运行期多会话:Pill 管对话,Tab 管文件

6 分钟。把"AI 会话管理"这件事跟"文件管理"分开 — 一个是药丸(Pill),一个是 Tab。

能力定位

解决什么

写作者常并行 2-3 件事:周报的草稿需要 AI 帮润色;另有一条 skill 调研跟同事相关;还有一份技术笔记在等 AI 帮你校对。 3 件事 = 3 个 AI 会话 = 互相不能串上下文(否则 AI 把"用户的语境"带到不相关的事里)。 普通 chatbox 只给你一个会话,要并行就需要你自己来回切。Typola 让这件事变简单。

Typola 怎么做

AI 工作台 header 有一个药丸按钮(💬 <当前会话名> ▾ <数量徽章>)。 点开 → 下拉菜单:列出会话列表 + 「新建」按钮 + 每条会话的「关闭」按钮。 药丸不像 Tab 占一整行,只是 header 里一颗圆角按钮,形态区别于编辑器顶部的文件 Tab — 一眼区分"文档是 Tab,会话是 Pill"。

跟别家不同

Claude Desktop 用 Tab、Coze 用大列表、Notion AI 用对话侧栏 — 都太重。 Typola 学的是 OpenDesign 的 "Workspace tabs bar + Conversations pill" 双管分工,克制到极致: 下拉只做切换/新建/关闭,不做重命名/搜索/归档(那是"把对话变档案库",会引向 Claude Desktop 的聊天工作台,Typola 不干这事)。

前置

  • 已完成第 2 课(知道 AI 工作台在哪)
  • 装好 CLI,检测通过

1. 看药丸按钮

进心流模式 → 看 AI 工作台 header(顶部):

┌─ 头部 ────────────────────────────────────────┐
│ 💬 <当前会话名> ▾ · N           (主操作区)    │
│   ↑ 你点的就是它                                  │
└──────────────────────────────────────────────┘

这颗药丸就是会话切换器。点开它 → 一个下拉面板:

  • 顶部:会话列表(当前会话前面有 ✓)
  • 列表底部:「+ 新建对话」按钮
  • 每个会话条目:右边有一个 ✕ 关闭按钮(当前会话不能关)

2. 新建会话

下拉面板里点「+ 新建对话」:

  1. 左栏立刻开新会话(无 prompt 先就空着)
  2. 输入框清空、Ready 状态显示
  3. 下拉面板列表多一条新的「自由对话」(M1 限定:只能以 skill 名开头的会话或自由对话 — 你也可以从前置 skill 直接开)
"自由对话" vs "skill 会话" 两种入口
  • 自由对话 = 你点了「+ 新建对话」开的会话,纯 prompt,不挂 skill
  • skill 会话 = 你从第 4 课 SkillHub 点某张 skill 卡开的会话,标题 = skill 名、UUID 独立

两种会话等价于"独立的草稿纸";管理方式一样,只是入口不同。

3. 并行 3 个会话,互不串味

写作者常态:你有 3 件事并行。

操作:

  1. 开第 1 个会话 — 起名叫「周报草稿」(自动)→ 问 AI 关于本周项目进展
  2. 点药丸 → 「+ 新建对话」开第 2 个
  3. 发 prompt 给第 2 个 — 例如「帮我看看 /huawei-style-ppt-skill 用什么场景」(skill 调试)
  4. 再开第 3 个 — 给同事写一段技术笔记标题

现在:

  • 药丸右侧出现数字徽章 3
  • 下拉面板里 3 条会话
  • 每条会话独立的 session UUID(独立 AI 上下文),互不串
验证"互不串" 的一种方法

在会话 A 里说"我是产品经理,正在写一份面向工程师的 PRD",然后切到会话 B 问"我应该用哪种语气"; AI 在 B 里不会把"产品经理 / PRD"带进来 — 它的 session 状态是干净的。 只有切同一个会话的过往对话,AI 才会接着说。

4. 关闭不用的会话

下拉面板里每行右边有 ✕(会话 1 个的时候,✕ 是禁用的 — 至少保留 1 个):

  • 点 ✕ → Typola 弹轻确认「关掉后会丢失该会话历史」
  • 确认 → 会话消失、列表少一行,药丸落到下一条活跃会话
  • 已经发给 AI 的 prompt 还在会话内存里,闭掉就没了 — 不持久化

关闭 ≠ 删产品。第 7 课会展开"AI 生成的文件去哪了" — 那是 产物中心的事,跟会话关闭无关。

5. 不做的事(护城河)

会话下拉里有几个功能故意不做:

不做 原因
重命名会话 标题已经自动 = skill 名 / 「自由对话」,再让你手动命名 = 在把会话当档案管
会话搜索 有了搜索你就开始把对话当文档存 — 文档才是产物
归档 / 标签 同上
AI 用量统计 让作者更关心"省 token"而不是"写好稿"

设计团队 每加一个对话侧功能,都要问一句:这服侍文档(成品),还是把对话变资产? 服务文档的加(产物回流 / 选区注入 / skill 调用);把对话变资产的不加。

"那我的会话会被永久留底吗?"

不会。Typola 只在本机内存里维护会话,关 Typola = 丢全部会话上下文。 你写过的 prompt、给到的反馈、AI 的回答都没了。

想要保留 AI 给你的东西 → 产物中心(第 7 课)才是这条路: AI 写出来的文件(报告、PPT、草稿)会在 .typola-output/<会话>/ 落盘, 那个文件就是你的"该保留的部分"。

试一试

6 分钟任务:开 3 个会话并行,验证互不串
  1. 打开任意文档,切心流模式。
  2. 点药丸 → 「+ 新建对话」开第一个会话,prompt 发"我是一个 PM,正在写 PRD"。
  3. 打开第二个,prompt 发"我是一个独立开发者,正在做 Web 项目"。
  4. 打开第三个,prompt 发"我是一个老师,正在准备一节课"。
  5. 三个都开了之后,分别切回去问"我是做什么的?",看每个会话是不是只记自己的角色,不串。
  6. 关掉其中 1 个,确认 ✕ 行为符合预期(只剩 2 个,徽章变 2)。

✅ 3 个会话并行 + 互不串 + 关闭正常 = 进入第 7 课:产物中心。 你会学到 AI 生成的文件落到哪、怎么在编辑器打开、怎么归档到工作区、怎么撤销覆盖 — 这是 Typola 最"实际"的一节。

陷阱

  • 药丸按钮不见了 — 你在阅读模式或检视模式。切到心流模式(Shift + A 或顶栏控件)。
  • 关掉最后一个会话又被自动新建 — Typola 不允许 0 个会话,所以最后一个不可关(✕ 禁用或变成"+ 新建对话"占位)。
  • 同一会话上下文"串味" — 这是设计,不是 bug。同一会话的过往对话 AI 都能看到,这是 Claude / OpenCode 的原生行为。
  • 重启 Typola 会话全没了 — 持久化属 M3+ 路线图,没排到。你可以靠 产物中心(第 7 课)保存 AI 实际生成的内容。
  • 不同会话切回去发现"内容跟上一次不太一样" — 因为多个会话间有独立的 session UUID,切回去读的是那次会话的最新状态,而不是全局同步。
