第 8 课

交付 + 离线降级

10 分钟。Typola 能"出"了 — 4 种导出形态各自适合什么场景,以及没网/没 AI 时 Typola 还能做什么。

能力定位

解决什么

一篇稿子写好了,接下来要"出" — 给同事、客户、读者、上传公众号… 不同受众要不同形态:PDF 给最终交付,Word 给会再编辑的同事,HTML 给网页/邮件,review.md 给协作者或 AI。 形态选错 = 多花两小时再排版。

Typola 怎么做

Typola 强制导出与编辑共享同一个 Markdown 源(markdownExportRenderer 是 Phase 4 共享桥接)。 4 种导出默认会把 Markdown 渲成自己的格式;导出按钮走 Rust 后端,完成后右上角 toast 通知,不打断写作。 没了 AI、没了网,Typola 仍然可以作为本地编辑器使用 — 这部分叫降级。

跟别家不同

很多编辑器的"导出 PDF"其实是在调用一个云 API,Typola 用本地隐藏 WebView2 + Page.printToPDF,全程离线。 富文本复制走 WechatPreviewPane(兼容公众号后台),不需要打开浏览器再复制粘贴。 降级不是 fallback 列表,而是"你的工作不被网络/AI 绑架"的产品承诺。

本课目录
1. PDF 导出 — 最终交付 2. Word — 给会再编辑的同事 3. HTML — 网页、邮件、富文本(公众号) 4. review.md — 协作者或 AI 5. 离线降级:无网、无 CLI 时还能干什么 6. 自定义导出预设 试一试 陷阱

1. PDF 导出 — 最终交付

场景:把这篇文档原样交付 — 不再被修改、直接发出去 / 归档 / 评审。

  1. 打开要导出的文档
  2. 按 Ctrl + P 或菜单 导出 → PDF
  3. Typola 用一个隐藏的 WebView2渲染 Markdown,然后调 Page.printToPDF(本地,不上云)
  4. 保存路径自动从当前文件路径派生,无需开对话框:
    • 当前文档已存 → 同目录 / 同主名加 .pdf
    • 当前文档未存 → 默认落用户下载目录
  5. 右上角 toast:「PDF 导出成功 / 失败」
PDF 不响应你应用的主题色

Typola 的暖米黄/深海/墨韵主题只影响编辑器;PDF 走自己的导出配色(预设)。 理由:别人读 PDF 时不一定知道你用什么主题。 想自定义 PDF 样式 → 「自定义导出预设」(本节最后会展开)。

2. Word — 给会再编辑的同事

场景:把稿子给同事,他习惯改 Word — 你不用帮他去 Word 里排版。

  1. 导出 → Word (.docx)
  2. 可选:先打开Word 纸张预览(右栏的 WordPaperPreviewPane)看 Word 视角下的样子
  3. 确认后导出,产物落 <源>.docx
  4. 同事用 Word 打开可以正常修改
.docx 不是 round-trip

Typola 生成的 .docx 是「导出副本」,不是你未来再 import 回 Typola 的双向通道。 同事改完 Word 给你,你最好拿回 .md 在 Typola 里继续改 — Word 修改痕迹别强迫 Typola 反向解析。

自定义导出预设:Wor在「设置」里有 "Word 导出预设" 编辑面板,内置 8 个 slot 默认开启,你可以上传 JSON 自定义版式(字号/页边距/标题字号/字体):

{
  "name": "团队周报样式",
  "page": { "margin": "2.5cm", "fontSize": "10.5pt" },
  "heading": { "h1": "16pt", "h2": "14pt" },
  "font": { "body": "Microsoft YaHei", "code": "JetBrains Mono" }
}

3. HTML — 网页、邮件、富文本(公众号)

3 种用法,3 个不同按钮:

按钮 产物 用途
导出 HTML .html 文件 放到自己网站 / 内网 wiki / 邮件附件
复制富文本 / 公众号样式 HTML 片段(放进剪贴板) 粘贴到 微信公众号后台 / 知乎 / 掘金的后台富文本编辑器,样式跟随渲染
在浏览器打开 HTML 系统默认浏览器启动 本地预览 / 截图归档 / 给同事看效果
公众号粘贴小技巧
  1. 先打开 WechatPreviewPane(视图 → 公众号预览)— 看样式
  2. 调整字号、颜色到公众号合规
  3. 点「复制富文本」
  4. 打开公众号编辑器,粘贴(Ctrl + V)
  5. 微调标题/封面图就发

不再需要 Word + 排版软件多次中转。

4. review.md — 协作者或 AI

这是第 5 课的输出。两种用法:

给人看
同事直接看 review 版 markdown,知道每段问题在哪;不需要装 Typola
给 AI 改
把 review.md 喂给任意 AI(Claude / GPT / Gemini / 自家 AI),它们看 blockquote 后的「请处理」知道要改什么

导出路径取决于上下文:

在哪里导出 文件名
检视模式右栏 <原文件名>.review.md(带检视版标记,不污染原 .md)
AI 改稿产物 <原 stem>.ai改{N}.md(N = 改了几次)
导出 review.md 后,原文档彻底不受影响

Typola 的检视意见是叠加在内存里的一层,导出的 .md 是这一层的快照; 关闭 Typola → 那份 .md 还在磁盘(你刚导出的),内存里的意见消失; 重新打开 .md → Typola 不会再去把意见解析回那份 .md,单向快照设计。

5. 离线降级 — 没网、没 CLI 时 Typola 还能干什么

这是 Typola 的底线承诺:所有功能必须在断网/无 AI CLI 下退化为可用。PRODUCT.md 设计原则 #4。

断网 → 你还有这些
  • 所有 Markdown 写作与编辑功能(阅读/心流/检视 三模式)
  • 本地文件操作(打开 / 保存 / 改后缀 / 拖拽)
  • 查找替换、文件大纲、文档统计、Find/Replace
  • 主题切换(本地样式)
  • 图片粘贴保存(本地)
  • WordPaper 预览 + Word/PDF 导出(本地 WebView2)
  • 底层终端(可在任意目录跑命令 — 但要 Windows 已装 terminal)
断网 → 你没有这些
  • 所有 AI 改稿(浮条、检视 → AI、AI 工作台对话)
  • SkillHub 的"未装时让 Claude 装"路径(它依赖 AI 工作)
  • 产物落盘以外的 watcher(但不关键 — 文件在磁盘就在)

6. 自定义导出预设

在「设置 → 导出预设」可以:

  • 管理 Word 预设(内置 8 个 slot 默认开启)
  • 管理 HTML 预设(同样的 slot 机制)
  • PDF 默认走标准预设,但允许改字体
预设的形态与限制
  • 是 JSON 文件,不是 LaTeX 模板 — 学起来轻
  • 设置面板里直接给实时预览
  • 8 个 slot 上限 = 防止"我做了 30 个预设然后忘"

试一试

15 分钟任务:一稿四产出
  1. 打开任意写好的文档(无 AI 也能做)。
  2. 导出 PDF 一次,在 导出物 同目录看到新 .pdf。
  3. 导出 Word 一次,确认 .docx 可用 Word/WPS 打开。
  4. 复制富文本,粘贴到一个临时文本文件,看 HTML 片段里有 <h2>/<blockquote> 等干净标签。
  5. 断网(关 wifi + 关以太网),看 Typola 是否还能写、还能导出 PDF、还能用本地终端。断网下不要去 AI 改稿,这是预期行为。

✅ 一稿四出 + 知道什么时候没 AI 也能干 = 9 节课全部完成 🎉。 从第 0 节到这里,你已经掌握 Typola 全能力:写、改、检视、交付、外加在哪一节暂停。

陷阱

  • PDF 导出卡死 — 通常是图片路径解析失败,或字体加载超时。Typola 走 fail-open 路径,等 30 秒一定出结果。
  • Word 导出排版乱 — 多半是表格/代码块超长。预设里调字号、改用更紧凑的版式。
  • 公众号粘贴后字体不动 — 公众号富文本编辑器会覆盖部分样式。你需要在后台再选一次默认字体。
  • 导出后没找到文件 — 看保存路径:已保存的文档导出到同目录,未保存的文档落到 Downloads。
  • 断网后 AI 改稿按钮变灰 — 是正常降级,不是 bug。换到无 AI 的工作流即可。

教程收尾

4 句口诀,再念一遍:

· Tab 管文件,Pill 管对话。
· 左栏是草稿,中栏是成品。
· AI 改稿先 diff,采纳再回写。
· 降级不离本地,产物总回文件。

你现在可以:

  • 看 速查表 — 一份 1 页的常用快捷键 + 浮条动作 + Skill 列表
  • 回头精修任何一节 — 总是可以再练一遍
  • 用任何工具问 AI 一两个问题 — 你是 Typola 真用户了,剩下的场景你试出来

最后提醒:Typola 是本地优先、文档为中心的编辑器。所有 AI 工作都不应该偷走你的文档。 如果你发现某个功能让你心里有点慌(怕被覆盖、怕被清理),那就是 Typola 没做好,直接告诉我们 — 文档是你的。
