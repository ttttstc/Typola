第 4 课

Skill 场景:报告 / PPT / HTML 的固定套路

10 分钟。把"调 skill"从手敲命令变成点选 — 三个场景、十几个预置。

能力定位

解决什么

你装了 claude --version 能跑、能调 skill,但手敲 /huawei-style-ppt-skill 这种命令很别扭 — 还得记每个 skill 的名字、出处、能干啥。 Typola 把本机装的所有 skill 按"你会用它做什么任务"分类,你只管选场景、选 skill。

Typola 怎么做

右栏 SkillHub 是一组"场景卡"(首批三个场景:报告生成 / PPT 制作 / HTML 制作)。 每个场景下挂着系统推荐 skill + 你手动加进来的 skill。 点已安装的 skill → 开一个独立会话(标题= skill 名,输入框预填 /<skill-name> ,独立 session UUID)。 每个 skill 一个独立上下文 — 这是 Typola 的护城河。

跟别家不同

Cursor / Claude Desktop 让你手敲 / 调命令(命令面板),Coze / Dify 给你通用 marketplace。 Typola 不做 marketplace(那是产品错位),也不做"全量本机 skill 列表"展示 — 它只按场景把高频用法浮上来, 想看全部或加新 skill 走 + 添加 skill 弹窗。本机 skill 不是 Typola 的资产,是 Claude/OpenCode 的 — Typola 只"引用",不"拥有"。

本课目录
前置 认识 SkillHub 三个场景 点已装 skill 开新会话 未装 skill → 让 Claude 装 添加本机 skill 到场景 Claude 和 OpenCode 分别怎么调 试一试 陷阱

前置

  • 已完成第 2、3 课
  • 本机至少装了 claude 或 opencode
  • (可选)~/.claude/skills/ 下有 1-2 个 skill 文件夹(没装也没关系,会演示"未装时让 Claude 装"的流程)

1. 认识右栏 SkillHub

唤出心流模式后(Shift + A),看右侧栏。 上半段是 SkillHub(场景卡),下半段会留给产物中心(第 7 课展开)。

SkillHub 默认是一级视图:

┌─ SkillHub ──────────────────┐
│  报告生成     PPT 制作      │
│  HTML 制作                   │
└──────────────────────────────┘

3 张场景卡(报告生成 / PPT 制作 / HTML 制作)。系统内置首批。

为什么只有 3 个场景?

首批覆盖写作者最常要 AI 干的"出物"任务。其它(写周报 / 写 PRD / 写书评)暂时归到"报告生成"里扩展 — 不要让"全量场景"代替你思考,场景卡是预置的、克制的。

2. 三个场景分别预置了什么

场景 预置 skill(本机已装时显示卡片,未装置灰)
报告生成 首版空模板 + 你手动加进本场景的 skill。第 0 版故意空着,等你 "用出来" 才好推荐。
PPT 制作 huawei-style-ppt-skill、guizang-ppt-skill、huashu-slides、baoyu-slide-deck
HTML 制作 frontend-slides

3. 点已装 skill 开新会话

点一下场景卡(例如「PPT 制作」)→ 进入二级视图,看到 PPT 场景里挂的 4 张 skill 卡。 点已装的那张(卡片非灰色)→ 发生 4 件事:

  1. 左栏 新会话建立(标题等于 skill 名,例如「huawei-style-ppt-skill」)
  2. 输入框预填 /huawei-style-ppt-skill (光标在末尾,你打 prompt)
  3. 独立 session UUID生成 — 这个 skill 的多轮上下文跟其他会话完全隔离
  4. Composer 底部出现 Provider pill(默认沿用当前 Provider,Claude → Claude)

直接在预填后面写任务:

/huawei-style-ppt-skill 基于我的本周总结生成一份 8 页汇报 PPT

发出去,AI 加载 skill 并按其内部流程生成 PPT(产物会落 .typola-output/<会话>/,第 7 课展开)。

为什么"每个 skill 一个独立会话"?

护城河。写过周报 + 改文档的人都有过这种惨痛: 用同一个 AI 会话跨任务,AI 把上一次的"项目背景"擅自带了进来,导致周报开头多了"我们"这种 没必要的语境。 Typola 强制每个 skill 一个新 session UUID,从根上切断泄漏 — 你的周报 skill、PPT skill、技术笔记 skill 永远不会串味。

4. 未装 skill → 让 Claude 装

PPT 场景里有 4 张预置卡,但你本机可能一个都没装。 这时候卡片置灰但仍然保留 — 你能看到"它能干什么"(系统推荐摘要),从而判断要不要装。

点未装的卡片:

  1. Typola 不直接动文件系统(设计红线 — Typola 不替你安装,以免装错/装到错位置)
  2. 而是把一条安装指令 prompt 发到当前 AI 工作台会话
  3. 指令格式大约是:请帮我安装 Claude skill:huawei-style-ppt-skill。用途:<系统推荐摘要>。来源:<系统配置里的安装路径,或"未提供来源">。安装后请确认该 skill 可以通过 /huawei-style-ppt-skill 调用。
  4. AI 用 CLI 原生的能力去装它(比如 mkdir + 克隆 git repo + 写 SKILL.md)
  5. 装完你回 SkillHub 刷一下(右上角 ↻),卡片就显示"已装",可以用了
为什么不让 Typola 直接装?

原因 1:安全 — 让第三方编辑器写用户目录外的文件系统 = 不可控的攻击面。 原因 2:溯源 — 你从对话里能看到 AI 是用哪条指令装的,以后想自己改路径也不用"猜 Typola 怎么装的"。 让 AI 用它原生能力干活,Typola 只"派活",这是「我司不替你装软件」的克制。

5. 添加本机已装的 skill 到场景

你可能自己装了几个 skill 不在系统推荐里,想加到「报告生成」或「HTML 制作」场景:
二级视图右上角有 + 添加 skill 按钮。

  1. 点「+ 添加 skill」
  2. 弹窗列出当前 CLI 本机已经认识的全部 skill(不只是 Typola 系统推荐的)
    — 你可以搜索关键字过滤
  3. 勾选你想要的那条 → 确认
  4. 返回该场景二级视图,看到多了你刚加的卡片(图标 可识别为"用户加"标签)
  5. 系统预置不可删(避免误删),用户加的可删(小卡片上有 ✕)
"添加"和"安装"是两回事
  • 安装 = 文件系统上多了一个 skill 目录(Typola 不做这个)
  • 添加 = 本机已有 skill,告诉 Typola "我想在报告生成这个场景里看到它"(只是 skill-hub.json 多一条用户配置)

Typola 的 skill-hub.json v2 只存用户数据,绝不复制 skill 内容进系统模板。 升级 Typola 也不会丢你的用户配置。

6. Claude 与 OpenCode 在 SkillHub 上有什么不同

同一个 SkillHub 卡片在两个 Provider 下默认都显示,但实际调用方式完全不同:

Provider skill 怎么调用 扫描什么
Claude Code prompt 里写 /skill-name ... 给 CLI,触发原生 slash 命令 ~/.claude/skills/ 下的所有 skill
OpenCode CLI 参数 --command <name>;prompt 文本作为 opencode run 的位置参数追加 ~/.opencode/commands/ + ~/.opencode/command/ + opencode.jsonc 里的 command 条目
"装了但两个 Provider 都看不到" — Provider 隔离读得懂

Claude 的 skill 和 OpenCode 的 command 不互通 — 它们存在不同的目录,定义格式也不一样。 你装了 Claude 的 ~/.claude/skills/foo,那它只在 Claude Provider 下能用; OpenCode 看到的是 ~/.opencode/commands/foo.md。 想两边都能用,各装各的。

试一试

12 分钟任务:第一次用 Skill 完整跑通
  1. 唤出心流模式,看右栏 SkillHub,确认「PPT 制作」场景里至少有 1 张卡是已装的(显示非灰色)。
  2. 点「HTML 制作」场景 → 点 frontend-slides(假设你装了)→ 看左栏是否新开会话 + 标题 = skill 名 + 输入框预填 /frontend-slides 。
  3. 在预填后面写「基于我的 note.md(本课)做一份 6 页 HTML slides」,发送。
  4. 等 AI 输出完。AI 会写一堆 html/css/js 出来,落到 .typola-output/<会话>/,右下会出现 chip — 这是第 7 课内容,但你可以先看一眼 chip 长啥样。
  5. 关掉这个会话,在「报告生成」场景点「+ 添加 skill」,在弹窗里随便加 1 个本机已有的 skill,看返回后卡片多了一条。

✅ 全部完成 = 进入第 5 课:检视 + 内联审阅。 这一课是真"长稿利器" — 你会用一节课把"先攒意见,再让 AI 改"完整闭环跑一次。

陷阱

  • 卡片都置灰 — 你装了 skill 但 Typola 读不到。常见:skill 装在了 ~/.opencode/commands/,当前 Provider 是 Claude(只读 ~/.claude/skills/)。换 Provider 或装到正确路径。
  • 点场景卡跳到 AI 工作台但没新开会话 — 多半是对应的 Provider 没工作。先在 设置 → AI 执行 测一下当前 Provider 的 CLI 检测。
  • 预填的 /skill-name 我不想用 — 删掉就好。预填只是方便,不强求。
  • 添加的 skill 没消失 — 用户加的应该可删。点卡片 ✕ 一下。
  • 想加"未装"的 skill 到场景 — Typola 不允许。先装再加(走第 4 节"未装 → 让 Claude 装"流程)。

下一步预告

第 5 课:检视模式 + 内联审阅:逐处采纳 AI 改稿。 这是真实写作者最常用的一节 — 切到 检视模式,给一篇长稿攒意见,一键让 AI 按意见改,然后在中间编辑器逐处采纳/拒绝(Y/N/⏎),全程不离开原文。
