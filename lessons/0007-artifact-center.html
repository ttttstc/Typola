第 7 课

产物中心:AI 写的东西回到你的文件

10 分钟。Typola 护城河落地的最后一公里:AI 用工具写出来的文件,回来在我的中间编辑器里打开。

能力定位

解决什么

"AI 帮我写了一份报告,但它困在对话框,我得复制粘贴才出来" — 真正让你能用 AI 高质量产出的最后一步是产物落地: AI 用工具(Write/Edit)创建的文件要能进我的文件结构,我能在编辑器里像打开任何 .md 一样编辑它, 而不是只能看预览。

Typola 怎么做

AI 的工作目录被焊死在 <工作区根>/.typola-output/<会话 ID>/。 Typola 不光监听这个目录(顺便用 watcher 兜底),还解析 Claude 的 Write/Edit tool_use 拿到精确路径 + 内容。 右下角出现文件名 chip;点 chip 在中间编辑器打开(.md → WYSIWYG,.html → 阅读模式预览), 不在右栏铺。 文件想长期留可以「保存到工作区」(自动改名避免冲突);想覆盖当前文档就点「合并到当前文档」(走第 5 课的内联审阅)。

跟别家不同

Claude Desktop 把产物叫 "artifacts",挂在对话侧,让你在对话里看;那是把对话当档案库的设计。 Cursor 把产物叫 "code changes",塞到右栏 diff 里;那是把文档当代码的设计。 Typola 既不把对话当档案,也不把产物塞边栏 — 产物 = 文档,进编辑器,这就是为什么 Typola 把"产物"放在 中间位置而不是右栏。

本课目录
前置 产物落到哪 右下角 chip 产物中心面板 在中间编辑器打开 保存到工作区 合并到当前文档(走审阅) 覆盖原文 + 撤销覆盖 试一试 陷阱

前置

  • 已完成第 2、4 课
  • AI 工作台能跑(CLI 检测通过)
  • 当前 Composer 工作目录已设置(初次启动让你选过,可以在 Picker 里改)

1. 产物落到哪:固定目录,不污染工作区

Typola 强制 AI 的生成类会话(点 skill 开的、改整文档的)的 cwd 在:

<AI 工作目录>/.typola-output/<会话 ID>/

看 AI 工作目录:Composer 顶部的 WorkingDir 标签。点它能换。默认是 Typola 第一启动让你选的那个目录。

"为什么 cwd 焊这么深"

两层理由:

  • AI 写"运行时产物"(报告、PPT)→ 应该只在 .typola-output/ 出现,不会污染你原本干净的 工作目录(docs/、posts/ 等不该被随便写)
  • 保护工作目录里现有的文件 → AI 不会因为路径写错就改你仓库里的文件

想"留"某个产物 → 用「保存到工作区」(本节下面会讲);不想留 → 关 Typola / 一段时间后自动清理。

.typola-output 应该加进 .gitignore

这是临时产物,不是要入库的代码。把这一行加进你工作目录里的 .gitignore: .typola-output/

Typola 在常见工作目录里有 hint,显式不强制。

2. 右下角 chip 是产物的入口

AI 用 Write/Edit 工具写完一个文件后:

  1. Typola 立刻解析 tool_use 的 file_path + content
  2. 编辑区右下角弹一个浮层(ArtifactToast)
    📄 新产物 weekly-report.md · 打开 | 关闭
  3. watcher 兜底监听(防止 AI 走其他路径写的文件被漏掉)
  4. 同时把这个产物加到产物中心的当前会话列表

点 chip 的「打开」→ 在中间编辑器开新 tab + 加载文件。

不同类型的产物 chip 图标
  • 📄 Markdown(在最右栏补 产物 表标签)
  • 📊 PPT(HTML 形式)
  • </> HTML / HTM
  • 📑 检视版(从 review 导出的 md)
  • 📝 AI 改稿(第 5 课产物)
  • 🎨 公众号(富文本复制产物)

3. 产物中心面板

想看全部 / 历史产物,右栏有「AI 产物」面板:

  • 顶部 tab:
    • 当前会话(默认)— 这个会话产出的所有文件
    • 全部产物— 当前工作目录下所有会话的产物(列表按时间排序)
  • 搜索框 + 类型筛选(markdown / html / review / ppt / 公众号 / 数据)
  • 每条 row 显示:文件名 · 类型 · 生成时间 · 操作(打开 / 比较 / 归档 / 删 / 覆盖 / 撤销覆盖)

4. 在中间编辑器打开 — 不在右栏铺预览

关键设计决策,重读三遍:

产物 = 文档。文档归文档位。

点开 chip → 中间编辑器开新 tab(像你打开任何 .md 一样):

  • .md → WYSIWYG 阅读模式,可编辑
  • .html → 走 HtmlPreviewPane 阅读模式(只读、不在中间编辑,渲染之后可点"在浏览器打开")
  • PPT HTML → 同上(只读 HTML,完美套 frontend-slides 等的输出)
不是右栏预览。右栏铺预览就是把产物当"附件 / 副产物",那是 chatbox 的设计;Typola 把产物当文档。

图示区别:

chipbox / Cursor 模式:  [文档]  |  [产物预览]    ← 文档边栏看产物,产物降级
Typola 模式:            [文档 tab:weekly-report.md]        ← 中间编辑,正常文档

5. 保存到工作区 — 长期留

在产物中心 / chip 「菜单」找「保存到工作区」:

  1. 弹目标目录选择(默认 <工作区>/ 根,可换子目录)
  2. Typola 自动改名避免冲突:已有 weekly.md → 新名 weekly-2.md(或加时间戳)
  3. 文件从 .typola-output/<会话>/ 搬运到工作区(不是复制)
  4. 产物状态从 done → archived
什么时候该"保存到工作区"
  • AI 写的报告 / 周报你要拿出来用 — 立刻归档
  • AI 写的草稿你打算改改后当终稿 — 归档后用中间编辑器编辑
  • AI 写的临时素材 / 实验产物 — 不用归档,关 Typola 后自动清理

6. 合并到当前文档(走内联审阅)

AI 写的产物 = 整篇 .md 内容?那很可能你想用它替换或合并当前编辑的草稿。

chip 旁或产物行菜单有「合并到当前文档」按钮:

  1. 触发第 5 课的内联审阅态(不是粗暴覆盖)
    • 原文 = 当前打开文档的全文
    • 新版 = 产物文件全文
  2. 进入内联审阅态后,Y/N/⏎ 逐处对比,改动地图同步亮
  3. 点「应用」→ 合并结果写回当前文档 → 文档变脏

这里的设计选择是 — 即便产物是"AI 已经写完整"的产物,也不直接覆盖原文。 你能控制每一处接受与否,跟第 5 课的产物写回走同一条管线。

"AI 全文重写"的产物也走内联审阅

有 skill output: review_inline 标签(如"全文重写"型 skill)→ 产物到达后直接进入审阅态,不必手动点合并。 这是 Typola 的 A 类入口(AI_DIFF_PREVIEW_SPEC §2.1)。

7. 覆盖原文 + 撤销覆盖

当你确定 AI 写的产物就是终稿、要直接替换当前文档(不想要审阅态),

有两种粒度:

动作 走法
覆盖当前文档 产物行点「覆盖」按钮 → 弹确认「原文将被替换,可在 5 分钟内撤销」 → 文档替换 + 产物 manifest 标记"已覆盖"
撤销覆盖 同一行点「撤销」按钮 → 文档回到覆盖前状态(产物 manifest 标记撤销)
覆盖是"少看一眼"模式 — 真要用慎重

默认用「合并到当前文档」(走内联审阅);确定完全没必要对比时再走「覆盖」。 撤销有时间窗(目前是 5 分钟),超时后撤销不可用。

试一试

12 分钟任务:4 种产物处理方式全跑一遍
  1. 心流模式,在 AI 工作台开新会话,prompt 发"给我写一份本周 mock 报告,标题是 weekly-mock.md,内容 3 段即可"。
  2. 等 AI 用 Write 工具写完文件,看右下 chip 出现 → 点"打开"在中间编辑器看产物。
  3. 产物行点「保存到工作区」,选工作区根目录 + 不改名,确认搬运成功(.typola-output/ 里文件没了)。
  4. 开新会话,prompt 发"把上面这份 weekly-mock.md 改写一下,内容大改",等产物写出来。
  5. 产物行点「合并到当前文档」 → 你编辑器里要选一份打开的稿件 → 进内联审阅态 → Y/N/⏎ 走走 → 应用。

✅ 全部跑通 = 进入第 8 课:交付 + 降级。 你会学到 4 种导出(PDF / Word / HTML / review.md)、富文本复制到公众号、无网络/无 AI 时的兜底。

陷阱

  • chip 不出现 — AI 没用 Write/Edit 工具,而是直接在前几条消息里把内容贴了出来。这种情况下产物没落地,得手动复制。
  • 产物落在奇怪路径下 — AI 自作主张写了绝对路径。Typola 仍然解析并显示,但文件位置不是 .typola-output/。归档时手动选目标。
  • 保存到工作区改名逻辑奇怪 — 默认按"产品目录是否存在同名文件"自动追加 -2 之类。改名规则在 services/artifacts/manifest.ts 可以改提示。
  • 产物 HTML 想在浏览器里看 — 走 HtmlPreviewPane 右下角「在浏览器打开」按钮,Typola 调用系统默认浏览器。
  • .typola-output 越来越大 — 定期清理;或用产物中心面板顶部的「清理过期」按钮。
  • 覆盖错了,撤销不可用 — 检查产物 manifest 状态;过 5 分钟就真没法撤了。要养成"先合并后覆盖"的习惯。
