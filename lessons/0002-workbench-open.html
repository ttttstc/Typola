第 2 课

AI 工作台开箱:Shift+A 与多轮对话

8 分钟。把 AI 从"选段级改稿"升级到"围绕整篇文档来回问答"。

能力定位

解决什么

整篇稿子写一半,你想问 AI"这一段论据够不够"、"开头能不能更吸引人"、"有没有逻辑漏洞" — 这些不能靠浮条的选段动作解决,因为你要把上下文作为整体拿出去。需要一个能围绕当前文档来回对话的工作区。

Typola 怎么做

左侧栏在"文件树"和"AI 工作台"之间切换(不并存)— 选了 AI 工作台,文件树收起让位,编辑器保持全高不被挤压。 Typola 用本机 CLI 跑 headless 流式协议(Claude Code 走 stream-json,OpenCode 走 JSON),所有工具调用、思考、正文、应用内卡片化展示 — 你不用开终端、不用手动回车。

跟别家不同

Claude Desktop / ChatGPT / Cursor 是"对话为中心",文档是附加品;Typola 反过来 — 文档是 C 位,对话是草稿。 所以 Typola 不做"会话存档库",不做"AI 用量统计",不做"跨会话搜索";只做"围绕当前稿子 + 当前文档附件"的少量并行任务。这也是为什么用 Pill 而不是 Tab 管对话(下节课展开)。

前置

  • 你已装好 claude 或 opencode,Typola 设置 → AI 执行 检测通过
  • 已经完成第 1 课(浮条能正常工作)
  • 至少有一份写了一半的文档在打开

1. 唤出 AI 工作台

两种方式 — 任选:

Shift + A
心流模式 ↔ 阅读模式来回切
顶栏三段控件
手动点"心流模式"

唤起后你会看到三件事同时发生:

  1. 左侧栏:文件树收起,弹出 AI 工作台(对话流 + 输入框)
  2. 编辑器:高度不变,继续全高展示当前文档(不被对话框挤)
  3. 右栏:SkillHub 出现(第 4 课展开)

再按一次 Shift + A 或点回"阅读模式",就回到只有文件树 + 编辑器的常态。

"心流模式"这个名字的意思

心流 = flow = 沉浸写作状态。Typola 不想让你被对话框的常驻存在打断,所以左栏只在需要时让出来; 它跟"AI 工作台"是同一区域,只是更强调"这是写作者的沉浸写作位"。 看到"心流模式"四个字,脑子里就是"左栏切到 AI 工作台"。

2. 发第一条消息

左栏底部输入框(Composer)就是入口。

  1. 在输入框写一句问题(例如「帮我找出这篇文章里论据最弱的 3 段」)
  2. (可选)勾上你想让 AI 看见的上下文芯片:当前文档 默认就在
  3. 点 发送 或 Ctrl + Enter
  4. 等 AI 流式输出 — 看到下方出现卡片就开始收到回应

发送后输入框下方会出现一条取消按钮,任何时候想中止 AI 当前输出就点它,等同于"放下笔"。

3. 读懂 5 种流式卡片

AI 输出不是一坨文本滚下来,而是按内容类型分卡片出现 — 你得能识别:

卡片 它是什么 你该怎么做
思考卡(Thought) AI 在真正打字前的内部推理。默认折叠,点开看灰色的文字流。 通常不用读,遇到 AI 输出诡异再看。
正文卡(Markdown) AI 给你的回答本身(打字机式逐字出现)。 读它。打字机滚动时还能左右滑或停止发送。
工具卡(Tool) AI 调用了 Read / Edit / Bash / Glob 之类工具。低关注度工具默认折叠。 折叠着就好;AI 在读你的文档是正常的。
Done 状态栏 一轮结束时的耗时 · 输出 token · 估算花费。 看一眼知道这轮成本。
错误诊断卡 AI 卡死 / 鉴权失败 / 模型无权限 — Typola 把 CLI 的退出码翻译成人话。 看诊断文本 → Retry 或按它给的指引修。
怎么识别"AI 现在还在想" vs "AI 已经放弃"

正常等待 → 流式卡片在追加(打字机还在滚)。 诊断卡 出现 → AI 已经停了,在告诉你为什么失败。 空白超过 8 秒 → 多半 CLI 卡死,可以按取消,再 Retry。

4. 切换 Provider(Claude Code ↔ OpenCode)

Typola 支持两个 provider,Composer 底部有一个小药丸按钮组(不是设置里的): 点 Claude 就切 Claude Code,点 OpenCode 就切 OpenCode。

切 Provider = 新会话

这是重要规则:切换 Provider 会弹确认「会开始新对话,旧会话上下文不会跨 provider 延续」。 为什么:Claude 的内部状态格式跟 OpenCode 不一样,跨 provider 续传只会让 AI 一脸懵。 想保留上下文就别切 Provider;想试不同模型,在同一个 Provider 下用它的 CLI 自带切换机制(例如 Claude Code 的 --model 参数)。

切完 Provider 之后,输入框、Done 栏、错误诊断卡的视觉风格也会跟着换 — Claude 用它原生 stream-json,OpenCode 用 JSON 协议,UI 形态相同。

5. 多轮对话 — 上下文延续

发第一轮后,继续在输入框打字再发第二轮,**Claude 会记得你上一轮说了什么**(它的 session UUID 由 Typola 内存维护)。

这是写作者最常用的模式 — 同一稿子来回迭代:

👤 : 我写了篇关于 X 的稿子,帮我看开头能不能更吸引人
🤖 : (输出建议)
👤 : 那段能不能换成更克制的语气?
🤖 : (在上一轮建议的基础上改)
👤 : 改完了,我整篇都在这,最后整体顺一遍
🤖 : (通读)
多轮的边界

同一运行期内:会话上下文保留(每轮发请求时 Typola 把 session UUID 给 CLI 续传)。
重启 Typola:会话丢失(内存 Map 不持久化)。后者是已知限制,跨重启的持久化在路线图里(M3+)。

试一试

8 分钟任务:跟 AI 来回三轮
  1. 按 Shift + A 唤出 AI 工作台。
  2. 把你正写的文档主题贴在 Composer,问「请用 5 个 bullet 给我列一下,普通读者可能误解的 5 处」。
  3. 等 AI 输出。等过程里不要点取消,看流式卡片是怎么分步出现的。
  4. 第二轮发:「挑一个最严重的展开讲讲,顺便给我一个修复方向」 — 验证 AI 真的记住了你第一轮问的稿子。
  5. 看 Done 状态栏的耗时和 token 数,估算下你这轮大概花了多少钱。

✅ 全部完成 = 进入第 3 课:上下文芯片 — 让 AI 看见附件 / MCP / Plugin。

陷阱

  • AI 一直"在想"不输出 — CLI 没起来或鉴权错。等 8-10 秒,如果还没出来基本是诊断卡的问题。
  • 左栏唤出后编辑器变窄 — 这不是 bug,是误判。Typola 的左栏切换是替换不是叠加;编辑器是全高的,只是左栏占了原本文件树的位置。
  • 切完 Provider 旧对话消失 — 正常的。每次切 Provider 重开会话。
  • Done 栏写着 0 token 但确实输出了一大段 — 流式 parser 在最后一次 step_finish 之前就关了。
  • Composer 打字特别卡 — 高频 text_delta 压力下,前端 rAF 节流应该够;真卡就在 设置 → AI 执行 里换 Provider 试。

下一步预告

第 3 课:上下文芯片:让 AI 看见你要的素材。 当前文档 + 附件 + .mcp.json + Plugin 目录 都会作为 prompt 的一部分喂给 AI。装好了,你就能让 AI 同时看你的稿子和参考资料一起改。

切片鉴权报错 / 想了解 stream-json vs JSON 协议差异 / 想问"为什么 Typola 不做会话存档"?直接问 AI。
