## 设计取舍

- **不引入 `@lezer/markdown` parseTree**：`findTableAt` 用 30 行正则即可定位表头 / 分隔行 / 列数，覆盖 95% 真实用法；Lezer 解析对单个命令太重。
- **`FormatAction` 扩联合**而非新增 `kernel.formatTable` 方法：现有菜单 / Toolbar / keymap 三处自然联动，0 运行时代价；未来 P1 行/列操作继续往联合里加。
- **不复用 Vditor 表格**：vditor 写作路径已废（AGENTS.md 明确"写作源码 / 所见模式都用 CM6"），`vditorFormatService` 仅补 no-op case 兜类型，不在 P0 范围。
- **不复现 `tableFormatCurrentTable`**：issue §5 的 `TableCommandBridge` 接口里没有这条，Plan agent 的建议但非 hard requirement，按 Ponytail 优先偷懒不实现。
