# Typola 用户指导手册

> 面向第一次使用 Typola 的中文写作者。按本手册完成“打开 → 写作 → AI 协作 → 检视 → 交付”后，你应能独立完成一篇 Markdown 文档，并得到可交付的 PDF、Word 或 HTML 文件。

## 先看结论：15 分钟验收路线

准备一个新文件 `demo.md`，按下面顺序操作。每一步的“看到什么”就是验收标准。

1. 新建或打开 `demo.md`，输入一个标题、两段正文、一个列表和一张表格。
2. 按 `Ctrl/Cmd+S` 保存。再次打开应用，文档仍能从最近文件恢复（如果设置中开启了“重新打开上次文件”）。
3. 选中一句话，使用浮条“润色”，看到结果卡后点击“采纳”。原文应能被一次撤销恢复。
4. 打开右侧“Word 预览”，切换一个预设；纸张预览应随正文变化。
5. 打开“检视模式”，选中一句话添加人工意见，再点击“AI 检视”。右栏应出现可定位的 AI/人工意见。
6. 点击“AI 改稿”，在 Diff 页面逐处选择“采纳/拒绝”，最后点击“应用”。应用前正文不得被直接改写。
7. 从工具栏“导出”选择 PDF 或 Word，观察右上角分阶段进度，保存后在目标文件夹找到文件。

如果 7 步都完成，基础写作、AI 改稿、检视和交付链路已通过。

## 1. 安装与第一次启动

### 1.1 安装包

- Windows 安装版：运行 `Typola_*_x64-setup.exe` 或 `.msi`。
- 免安装版：解压 `Typola_*_windows-x64_portable.zip`，运行 `Typola.exe`。
- Typola 需要 Microsoft Edge WebView2 Runtime。安装包会在缺失时给出引导。
- PDF 导出还需要系统已安装 Google Chrome、Chromium 或 Microsoft Edge。Word 导出不需要 Pandoc、LibreOffice 或其他外部转换器。

### 1.2 AI 是可选项

不配置 AI CLI 也可以正常写作、预览和导出。如果要使用 AI：

1. 按 Claude Code 或 OpenCode 官方方式安装 CLI。
2. 在终端确认 `claude --version` 或 `opencode --version` 有输出。
3. 打开 **设置 → AI 执行**，为对应运行时填写可选路径并点击“检测”。

Codex 当前只提供检测卡片，不在 AI 工作台中发送请求。

## 2. 认识界面

Typola 的中心永远是当前 Markdown 文档，其他区域按需打开：

```text
┌ 工具栏：文件 / 编辑 / 预览 / 导出 / 设置 ───────────────┐
├ 文件树（可收起） ── 中央编辑器 ── 右侧预览或 AI 工作台 ┤
└ 底部终端（按需打开）───────────────────────────────────┘
```

- **中央编辑器**：默认是 CM6 写作视图；它保存真实 Markdown source。
- **文件树**：打开一个目录后浏览 Markdown、HTML、Word 文件；文件标签页之间切换不会丢失各自脏状态。
- **右侧面板**：可以是 Word 纸张预览、HTML 预览、SkillHub、检视工作台或 AI 产物中心。
- **底部终端**：运行 Claude Code/OpenCode 等本地命令，适合需要完整 CLI 交互的高级用户。
- **顶部模式切换**：阅读模式、心流模式、检视模式。三者切换不会改变 Markdown 内容。

## 3. 第一次写作

### 3.1 新建、打开和保存

- 点击“新建文档”，或按 `Ctrl/Cmd+N`。
- 点击“打开文件”，或按 `Ctrl/Cmd+O`，支持 Markdown、HTML、Word。
- 点击“打开文件夹”，或按 `Ctrl/Cmd+Shift+O`，选择并建立工作区；文件按需从左侧文件树打开，不会批量生成标签页。
- `Ctrl/Cmd+S` 保存当前文件；`Ctrl/Cmd+Shift+S` 另存为新文件。
- 文档标题栏双击可重命名已保存文件。未保存文件需要先保存。

自动保存是设置项，默认关闭。第一次使用建议保持关闭，先熟悉显式保存和撤销。

### 3.2 写作模式与源码模式

- **阅读模式**：以成品排版为主，适合通读。
- **心流模式**：打开 AI 工作台/场景/终端等协作区域，适合边写边调用工具。
- **检视模式**：把正文和检视工作台并排，适合集中处理意见。
- 点击工具栏源码图标或按 `Ctrl/Cmd+Alt+S` 切换源码模式。源码模式显示 Markdown 标记、行号、折叠和实时预览能力；再次按下返回写作视图。

常用格式按钮会直接修改 Markdown source，并进入同一份撤销历史：加粗、斜体、链接、引用、无序/有序列表、任务列表和格式刷。

### 3.3 表格

1. 把光标放在文档中，点击“插入表格”。
2. 在单元格内右键，使用中文表格菜单插入、移动、复制、清空或删除行列，也可以设置对齐。
3. `Tab` 移到下一个单元格，末尾继续按 `Tab` 会追加一行；`Enter` 进入下一行。
4. 表格操作进入原生撤销历史，`Ctrl/Cmd+Z` 可回退。

粘贴 Markdown、TSV/CSV 或 HTML 表格时，Typola 会尽量转换为 Markdown 表格；转换失败时保留原文本，避免静默丢数据。

### 3.4 图片、公式和 Mermaid

- 图片可通过工具栏“插入图片”、拖拽或粘贴进入文档。
- 默认复制到当前文档目录下的 `assets/`，并优先写相对路径。
- `![alt](path)` 的本地图片会在编辑器、Word/HTML 预览和 PDF 中解析；网络图片保持远程地址。
- `$$...$$` 或行内 `$...$` 显示 KaTeX 公式。
- ```` ```mermaid ```` 代码块在可视区域渲染成图；语法错误时保留源码并显示错误卡片，不会阻塞其他段落。

图片规则可在 **设置 → 图像** 调整：保持原路径、复制到指定文件夹、上传命令、相对路径、`./` 前缀和 URL 转义。上传命令失败会自动回退到本地复制。

## 4. 预览与交付前检查

### 4.1 Word 纸张预览

点击工具栏 Word 图标，或按 `Ctrl/Cmd+Alt+P`。右侧显示接近 Word 的纸张分页效果。

- 在预览顶部选择 Word 预设，预览会即时更新。
- **设置 → Word 导出** 可启用/停用预设、导入 JSON v2 预设或在自定义槽位保存配置。
- 预览重点检查：标题层级、页边距、表格宽度、图片比例、代码块和页码。
- `.docx` 文件可打开查看，但当前不支持把已打开的 Word 文件再次导出为 Word/PDF；请先另存为 Markdown 或 HTML。

### 4.2 HTML 预览与复制

点击 HTML 图标，或按 `Ctrl/Cmd+Alt+M`。

- HTML 预览使用 HTML 导出预设，适合检查标题、引用、表格、代码和公众号样式。
- **设置 → HTML 导出** 可选择内置 CSS 预设，导入/导出 CSS JSON，或在自定义槽位保存样式。
- 在 HTML 预览中点击复制按钮，可复制富文本 HTML 到发布平台；点击导出可生成独立 `.html` 文件。
- AI 生成的 HTML 产物在“AI 产物中心”中会以沙箱 iframe 预览；“源码”按钮才会打开到中央编辑器。

### 4.3 PDF 预览检查

PDF 使用统一 Markdown 导出 renderer，不读取编辑器 DOM。导出前建议先打开 Word 或 HTML 预览，确认内容，再按 `Ctrl/Cmd+P` 或 **导出 → PDF**。

导出流程会显示：选择保存位置 → 解析 Markdown 与本地资源 → 页面排版 → 生成 PDF 页面 → 文件写入。取消保存不会启动昂贵的渲染；浏览器未找到时，安装 Chrome/Chromium/Edge 后重试。

## 5. AI 工作台：从零完成一次高阶协作

### 5.1 配置 Provider 和工作目录

1. 打开 **设置 → AI 执行**，检测 Claude 或 OpenCode。
2. 选择一个可用运行时作为活动 Provider。Claude 和 OpenCode 都可以发送；切换 Provider 会开启新的 Provider 会话，不会混用上下文。
3. 打开心流模式（点击模式切换器，或按 `Ctrl/Cmd+Shift+A`）。
4. 在 AI 工作台底部打开工作目录选择器，选择项目根目录。工作目录决定 CLI 的 cwd、产物目录和上下文文件范围。

工作目录没有选择时，Typola 使用用户默认目录；它不会偷偷把文件树目录当成 AI 工作目录。

### 5.2 发送第一条请求

1. 在 Composer 输入：`请概括当前文档的目标读者、结构和三个主要问题。不要修改文件。`
2. 确认底部 Provider、模型和工作目录显示正确。
3. 按 `Ctrl/Cmd+Enter` 或点击“发送”。
4. 观察助手正文、思考/工具卡和完成状态。长任务可以点击“停止”。

当前文档会以上下文芯片出现。拖入其他 Markdown、图片或参考文件作为附件；发送时它们会作为本地路径上下文传给 Provider。

### 5.3 Provider 切换和会话管理

- Composer 底部 Provider pill 可在 Claude Code / OpenCode 之间切换。
- 切换 Provider 会取消当前运行，启动新的 Provider-bound Conversation；旧会话仍可从会话 pill 切回。
- 会话可以新建、重命名、关闭；切换文件不会清空通用对话历史，但会重新注入当前文档上下文。
- Claude 支持插件目录和 `.mcp.json` 编辑入口；OpenCode 使用其自身的 command 配置，不要把 Claude `/skill` 命令直接复制过去。

### 5.4 SkillHub：把高阶流程变成可复用入口

1. 在心流模式右侧打开 SkillHub/场景面板。
2. 先确认当前 Provider，再扫描本机能力。
3. 选择一个系统场景（例如报告、PPT、公众号或写作辅助）；如果场景缺能力，点击“添加到场景”。
4. 可从本机已安装 Skill/command 中搜索添加，也可手动填写名称。
5. 点击场景卡后，Typola 会把能力预填到 Composer；检查当前文档、附件和工作目录后再发送。

Claude 以 `/skill-name` 形式调用；OpenCode 以 command 形式调用。SkillHub 会按 Provider 过滤，不会把不兼容的能力显示成可发送项。

### 5.5 产物中心：让 AI 文件回到本地

AI 生成的 Markdown、HTML、PPT 草稿、检视版和图片等文件会进入当前工作目录的 `.typola-output/<conversation>/`。

- 工具栏点击“AI 产物”，切换“当前会话 / 全部产物”。
- 用搜索框按标题、文件名或 Agent 筛选；用类型下拉筛选 Markdown、AI 改稿、检视版、HTML、公众号或 PPT。
- Markdown 点击“打开”会进入中央编辑器；HTML 点击“预览”在右侧沙箱中打开；图片使用系统图片工具打开。
- AI 产物可以“对比”“所在文件夹”“归档”“删除”。可覆盖原文的产物会显示“覆盖原文”，覆盖后可以“撤销覆盖”。

建议先预览/对比再覆盖原文。覆盖和归档都有路径保护，不会因为一个错误的产物路径改写任意文件。

## 6. 检视模式：从意见到可控改稿

### 6.1 添加人工意见

1. 切换到“检视模式”。
2. 在正文选中需要修改的句子或段落。
3. 点击选区浮条“加检视意见”，输入具体、可执行的建议，例如“把这段拆成三点，并保留原有数据”。
4. 保存意见。右栏列表会显示原文片段、意见、来源和定位行号。

浮条可以在 **设置 → 编辑器 → 浮条可见性** 关闭；关闭后仍可用右键菜单或 `Ctrl/Cmd+K`。

### 6.2 AI 检视

在右栏“AI 检视”中按需填写：

- **规则文件**：导入一个或多个 Markdown 写作规范。
- **写作规范 Skill**：勾选当前 Provider 可用的 Skill。
- **手工规则**：补充本次检查重点，例如“重点检查重复论证和报告腔”。

点击“开始检视”后，AI 只生成意见，不直接改正文。运行中可点击“停止检视”。完成后检查新增意见数量，无法可靠定位的意见会被跳过并提示原因。

### 6.3 处理意见和发起 AI 改稿

- 列表可按“全部 / 人工 / AI / 已忽略”筛选。
- 点击意见卡片可编辑、定位原文；“忽略”不会删除意见，之后仍可从“已忽略”恢复。
- 确认有效意见后点击底部“AI 改稿”。默认只把未忽略意见发送给当前 Provider。

AI 会生成候选稿，不会直接写回正式文档。进入 Diff 页面后：

1. 左侧“修改前”只读，右侧“候选稿”可编辑。
2. 用“上一处 / 下一处”或 `Shift+Enter / Enter` 浏览差异。
3. 每处点击“采纳”或“拒绝”；也可以“全部采纳 / 全部拒绝”。
4. 需要补充要求时，在底部输入反馈，选择“当前差异 / 当前章节 / 全部候选稿”，点击“继续修改”。
5. 确认自检状态为“自检通过”或可接受的警告，再点击“应用”。

只有点击“应用”才会写回当前文档；应用前可以“另存为”保留候选稿。若源文档在候选稿生成后发生变化，系统会阻止直接应用，选择“以最新文档重新开始”或“保留候选稿并重新对比”。

### 6.4 历史和检视版

- “应用前历史”保留应用候选稿前的版本，可与当前文档对比或以 Diff 方式恢复。
- “导出检视版”会另存一个带可恢复意见元数据的 Markdown，不覆盖源文档。
- 关闭 Diff 或切换文档前，若候选稿有未应用修改，Typola 会提示确认，避免误丢。

## 7. 图片与资源的高阶用法

打开 **设置 → 图像**：

- **保持原路径**：适合已经有稳定资源目录的项目。
- **复制到指定文件夹**：默认 `assets`，支持 `{filename}`、`{year}`、`{month}`，适合按文档或日期归档。
- **上传图床（自定义命令）**：命令输出最后 N 行 URL（每张图片一行）；点击“验证”先用一张图片测试。
- **优先使用相对路径**：推荐开启，便于移动整个文档目录。
- **相对路径加 `./` 前缀**：需要兼容特定静态站点时开启。
- **插入时转义路径**：文件名包含空格或特殊字符时可开启。
- **允许 YAML 自动上传**：开启后，文档 frontmatter 写 `typora-copy-images-to: upload` 才会触发上传。

上传命令失败会回退到本地复制；文档不会因为一次网络或命令失败而丢失图片。

## 8. 外观、字体和可读性

打开 **设置 → 外观**：

- **自定义模式**：点击顶部画笔按钮，用 Hue Wheel、饱和度、渐变和图案组合界面颜色；没有选择时使用纯白。
- **主题模式**：选择素笺、深海、墨韵、抽象或粗野主题。主题只改变界面和编辑辅助色，不改变 Markdown 内容。
- **检视模式增强 AI 改动与标注颜色**：长文检视时建议开启。
- **编辑器纸纹**：只对支持的主题启用静态纹理；深海与抽象主题不启用。
- **界面缩放**：80%–120%，适合高分辨率屏幕。

预览字体、字号、行距和宽度在 **设置 → 预览**；Word/HTML 导出样式仍由各自预设控制，不会因为切换应用主题而改变。

## 9. 终端和离线工作

- `Ctrl/Cmd+`` 打开或关闭终端；`Ctrl/Cmd+Shift+`` 新建终端标签。
- 终端默认使用 `pwsh.exe`、`powershell.exe` 或 `cmd.exe` 中可用的一项。
- AI CLI 在本机可用时，终端适合需要交互授权、查看完整日志或执行项目命令的场景。
- 无网络时仍可编辑、保存、预览、使用本地图片和导出 Word；AI/图床/远程图片可能不可用。
- PDF 导出依赖本机浏览器，不依赖网络下载；若浏览器已安装，离线也可打印。

## 11. 常见问题

### PDF 提示找不到渲染器

安装或更新 Google Chrome、Chromium 或 Microsoft Edge，完全退出并重新打开 Typola 后重试。Typola 不会自动下载浏览器。

### Word 导出很慢或没有进度

确认没有同时进行 PDF 导出；等待右上角 toast 完成“解析 Markdown → 生成 Word 版式 → 打包 → 写入”。首次导出会加载 `docx` 运行时代码，后续导出通常更快。Word 不需要 Pandoc。

### AI 运行失败

先到 **设置 → AI 执行** 点击对应 CLI 的“检测”。检查路径、模型字符串、登录状态和工作目录权限；OpenCode 与 Claude 的配置不能互换。没有 AI CLI 时仍可继续本地写作和导出。

### 图片显示为空白

确认图片路径相对于当前 Markdown 文件，或重新通过“插入图片”处理；移动文档时把 `assets/` 一起移动。远程图片需要网络和目标站点允许加载。

### 为什么 AI 改稿没有直接改正文

这是安全设计：AI 先生成候选稿，必须在 Diff 页面逐处审阅并点击“应用”。候选稿可以另存，源文档在应用前保持不变。

### 右侧预览太窄

拖动中央编辑器与右侧面板之间的分隔线；Word 预览会在较窄宽度下保持纸张贴近分隔线，减少空白。

## 12. 完整能力验收清单

小白用户在没有阅读源码的情况下，完成以下项目即可视为“主要功能通过”：

- [ ] 打开、新建、保存、另存为和重新打开 Markdown。
- [ ] 在写作模式和源码模式之间切换，完成一次格式化、撤销和查找替换。
