# AI 检视模式页面交互补充规格

> 本文是 `AI_REVIEW_DESIGN.md` 的规范性补充，覆盖统一检视意见列表、AI 检视入口、意见忽略、检视版导出与回读、正文行号和意见回链。

## 1. 设计目标

检视模式同时承载人工检视和 AI 检视。两类意见共享文档位置、意见内容和处理状态，但不强行共享问题分类。人工意见保持用户原始表达；AI 意见说明检视依据。检视版以新 Markdown 文件导出，不覆盖源文件，并能被另一台 Typola 回读为可视化意见列表。

## 2. 统一意见列表

右侧检视面板使用一个列表展示人工意见和 AI 意见，默认按照文档位置排序。

列表顶部只提供四个筛选：

```text
全部 | 人工 | AI | 已忽略
```

每条意见的公共信息只有：

- 文档锚点；
- 意见内容；
- 来源；
- 处理状态。

人工意见显示“人工”来源标签，不要求用户填写问题类型、严重程度或置信度，系统也不调用 AI 猜测这些属性。

AI 意见显示“AI”来源标签，并可以附带 `style.md`、Skill 或本次用户要求等依据来源。AI 问题类型是可选展示信息，不是统一列表的必填字段。

## 3. AI 检视入口

AI 检视从现有右侧检视面板发起，使用轻量设置面板：

```text
AI 检视

检视依据
✓ 当前项目 style.md
Skill：[可选 Skill]

本次要求
[重点检查重复论证和报告腔]

[开始检视]
```

交互规则：

- 自动识别可用的 `style.md`；
- Skill 可选，不要求用户每次选择；
- 本次要求使用自然语言输入；
- 不向用户展示拼装后的完整 Prompt；
- 已有人工意见可以提供给 Agent 作为上下文和去重依据，但不得被 AI 改写或分类；
- AI 检视结果直接加入统一列表并标记为未处理，不增加“导入列表”的二次确认。

## 4. 忽略意见

“忽略”是可恢复的处理状态，不是意见类型，也不等于删除。

每条意见的更多菜单提供：

```text
定位原文
编辑（人工意见）
忽略 / 恢复
删除
```

忽略后的行为：

- 从默认列表移入“已忽略”筛选；
- 不参与后续候选稿生成；
- 不进入默认检视版导出；
- 可以随时恢复；
- 不改变原文和意见锚点。

人工意见和 AI 意见使用相同的忽略状态。人工意见仍可直接编辑或删除；AI 意见只允许忽略、恢复或删除。

## 5. 检视版导出

导出检视版始终创建新文件，不覆盖或修改源文件。文件名沿用“原文件名-检视版N.md”的递增规则。

导出内容由两部分组成：

1. 源文档正文；
2. 追加在对应段落下方的未忽略意见。

人工意见保持原文，不被 AI 改写或补充类型。AI 意见可以在可读标题中显示依据来源。

普通 Markdown 阅读器应当能够直接阅读正文和段落下方的意见。Typola 专用元数据不得影响普通 Markdown 渲染。

## 6. 可回读检视格式

检视版使用版本化的 Typola Review 标记。文件级标记用于识别检视文档；每条意见使用不可见 HTML Comment 保存机器可读元数据，并在其后保留人类可读的 Markdown 意见块。

每条意见的机器可读信息至少包括：

- 意见唯一标识；
- 来源是人工还是 AI；
- 原文片段；
- 前缀或上下文提示；
- 导出时的 Markdown 源行号；
- 意见正文；
- AI 意见的可选依据来源。

文件格式必须满足：

- 普通 Markdown 工具忽略元数据并正常展示正文和意见；
- 元数据缺失或损坏时，文件仍可作为普通 Markdown 打开；
- Typola 打开文件时识别“包含检视意见”，但不强制切换模式；
- 用户进入检视模式后恢复统一意见列表；
- 点击意见定位并高亮检视版中的对应原文。

## 7. 意见定位与回链

意见定位优先使用现有文本锚点能力：

```text
原文片段
前缀提示
上下文恢复
导出时行号辅助
```

Markdown 行号只用于显示和人工沟通，不作为意见的唯一锚点。文档编辑导致行号变化后，仍优先通过文本锚点恢复位置。无法定位时，意见保留在列表并显示“原文位置已变化”，不得跳到错误段落。

## 8. 正文行号

正文右键菜单增加“显示行号”视图开关。该设置不写入 Markdown 内容。

行号定义为 Markdown 源文件的实际行号：

- 源码模式沿用源码编辑器行号；
- 所见即所得模式只在标题、段落、列表项、引用块和代码块等 Markdown 块的起始位置显示源行号；
- 视觉换行不产生新的行号；
- 检视意见列表可以显示对应的源行号；
- 点击意见仍使用文本锚点定位，行号仅辅助回链。

## 9. 用户路线

### 人工检视

```text
进入检视模式
→ 选中文字并添加人工意见
→ 意见进入统一列表并标记“人工”
→ 点击意见返回原文
→ 导出检视版或参与 AI 改稿
```

### AI 检视

```text
进入检视模式
→ 选择 style.md、Skill 和本次要求
→ Agent 在当前 Session 中执行一轮检视
→ AI 意见进入统一列表并显示依据来源
→ 用户忽略、保留或用于生成候选稿
```

### 协作者回读

```text
接收检视版 Markdown
→ 使用 Typola 打开
→ 看到“包含检视意见”的提示
→ 进入检视模式
→ 恢复统一意见列表
→ 点击意见跳到对应原文
```

## 10. 验收场景

- 人工意见和 AI 意见在同一列表中按来源筛选；
- 人工意见不要求类型，AI 不会自动为其分类；
- AI 检视可以组合 `style.md`、Skill 和本次要求；
- 忽略意见后，默认列表、候选稿生成和检视版导出均不再包含该意见；
- 恢复意见后重新参与列表、改稿和导出；
- 导出检视版不会修改源文件；
- 普通 Markdown 阅读器能正常看到正文和段落下方的意见；
- Typola 能识别检视版并在检视模式中恢复意见；
- 点击意见能够通过文本锚点定位并高亮原文；
- 文本锚点失效时不发生错误跳转；
- 右键行号开关不改变 Markdown 内容；
- 所见即所得模式显示的块起始行号与 Markdown 源行号一致。

## 11. Out of Scope

- 强制要求人工意见填写分类；
- 使用 AI 猜测人工意见类型；
- 将已忽略意见默认导出；
- 将检视意见写回或覆盖源文件；
- 使用单独的 sidecar 文件保存检视意见；
- 把 Markdown 行号作为唯一定位依据；
- 打开检视版后强制切换到检视模式；
- 自动合并发生位置冲突的检视意见。
