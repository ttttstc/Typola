# Product

## Register

product

## Users

中文写作者 — 已经在本机配置 Claude CLI、有自己的 AI 流水线、习惯用 Markdown 交付文档的人。不是「AI 重度使用者」,而是「把 AI 当笔的写作者」:产品经理、技术布道、独立研究员、公众号/小红书长文作者、准备日报周报的从业者。

使用场景:坐在办公桌前或深夜写稿,屏幕一开就是一两个小时。打开 Typola 通常是因为有一篇要写、要审、要交给协作者,而不是「想试试这个工具」。他们已经熟悉 Markdown,有自己的写作惯例,讨厌每个工具都重新学一套交互。

## Product Purpose

把 Typola 做成「文档即产物」的本地工作台 — 文档本身是从写作者的 AI 流水线里产出的产物,而不是 AI 替你写的成稿。Typola 把「阅读 / 心流 / 检视」三种工作状态压缩进同一个工作区,选区级 AI 操作(润色、缩写、扩写、术语解释)不离开编辑器,产物直接落回文档,AI 修改可一键撤销。

成功意味着:写作者打开 Typola 之后没有任何一次「切到 ChatGPT → 复制 → 粘贴回来」,所有 AI 调用都发生在他正在写的文档里。导出 PDF / Word 不需要重新粘贴到 Word 排版,所见即所得的纸张预览就是终稿。

## Brand Personality

- 沉静 — 不抢戏、不喧哗、不用 hero 数字、不用情绪化渐变
- 编辑感 — 像打开一本排过版的书,而不是像打开一个 SaaS 仪表盘
- 在场 — 工具退后,文字在前;强调色的作用是「这一处需要你注意」,而不是「这一处很酷」

3 个词:**沉静 · 编辑 · 在场**。

参考方向:Substack 的早期编辑台、Medium 排版最讲究的年代、Apple Notes 的克制、Stratechery 的长文阅读体验。重点不是「像哪一个」,而是「让人意识到字本身是被尊重的」。

## Anti-references

1. **「AI 替你写」式对话型 IDE**:ChatGPT / Claude Desktop 风格的对话面板、亮紫渐变、大段自动补全气泡。Typola 的 AI 是扩音器,不是代笔。
2. **通用 SaaS 紫蓝渐变 hero**:Notion 彩虹 hover、Linear 极客紫、VS Code 蓝调、Stripe 风的渐变 hero。Typola 不卖 SaaS。
3. **2026 AI 默认「暖米黄 + 衬线」样板**:饱和的暖米黄底 + Iowan/Source Serif 标题 + eyebrow kicker + 数字编号 section + rounded card 网格 — 这是当下训练数据的高频反射,Typola 要避开它。
4. **极简纯白 Markdown 编辑器**:Typora 那种「白底黑字、最大字号、最大留白」。Typola 是工具,不是纯白笔记本,有温度但有结构。

## Design Principles

1. **写作者第一, AI 第二**。任何界面决策先问「这帮写作者多想了一秒了吗」,再问「这看起来酷吗」。
2. **文档即产物**。预览视图就是终稿的样子;导出文件不能再二次排版。
3. **现有 token 是品牌锚点, 不动底色**。暖米黄 + 暖橙 + 衬线是 Typola 的视觉签名;逃离 AI 样板靠版式、字距、字体配对、构图节奏,而不是靠换底色。
4. **本地优先**。所有功能必须在断网/无 Claude CLI 下退化为可用 — 不依赖云、不强制登录。
5. **键盘可达 = 默认**, 不是高级选项。每一个交互都有快捷键,鼠标不是唯一入口。

## Accessibility & Inclusion

- 目标:**WCAG 2.1 AA**(对比度 ≥4.5:1 文本 / ≥3:1 大文本,键盘可达,焦点可见)
- 必须尊重 `prefers-reduced-motion`,所有动效有静态回退
- 强调色与成功/危险色不能仅靠色相传递状态(需配合 icon 或文字)
- 中文优先,CJK 字体在所有 fallback 链中靠前
